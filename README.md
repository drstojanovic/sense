    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-sense" class="anchor" href="#sense" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>sense</h1>
<p>Sense is a sensing and monitoring application for Android with a wide variety of sensors (called trackers from this point on) and ways to visualize the collected data.</p>
<p>Implemented trackers:</p>
<ol>
<li>Activity Tracker (using Google's Activity Recognition API)</li>
<li>Battery Tracker</li>
<li>Call Log Tracker</li>
<li>Camera Tracker</li>
<li>Microphone Tracker</li>
<li>Running Applications Tracker</li>
<li>SMS Tracker</li>
<li>Screen Tracker</li>
<li>Steps Tracker</li>
<li>WiFi Tracker</li>
</ol>
<p>Implemented visualizations:</p>
<ol>
<li>Text</li>
<li>Bar Chart</li>
<li>Pie Chart</li>
<li>Line Chart</li>
<li>List View</li>
<li>Map</li>
</ol>
<p>Sense consists of 4 main activities:</p>
<h3><a id="user-content-dashboard" class="anchor" href="#dashboard" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Dashboard</h3>
<p>Used to visualize the collected data for the day. Since trackers can have multiple visualizations, each of them can be shown on the dashboard.
Clicking on a dashboard card opens up a visualization activity for that tracker.</p>
<p><a href="https://camo.githubusercontent.com/f9ddd3c0273cb16eaab66721362ed574fc0dd057/687474703a2f2f692e696d6775722e636f6d2f6343463937597a2e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/f9ddd3c0273cb16eaab66721362ed574fc0dd057/687474703a2f2f692e696d6775722e636f6d2f6343463937597a2e706e673f31" alt="Dashboard" data-canonical-src="http://i.imgur.com/cCF97Yz.png?1" style="max-width:100%;"></a></p>
<h3><a id="user-content-visualization" class="anchor" href="#visualization" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Visualization</h3>
<p>Shows available visualizations as tabs, and attributes which can be visualized as a dropdown. Users can choose a date range as a filter, in which case all data in that period would be visualized.</p>
<p><a href="https://camo.githubusercontent.com/47632c4f414bbe4e87c7529c702225ee486c590f/687474703a2f2f692e696d6775722e636f6d2f365645386e55312e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/47632c4f414bbe4e87c7529c702225ee486c590f/687474703a2f2f692e696d6775722e636f6d2f365645386e55312e706e673f31" alt="Visualization" data-canonical-src="http://i.imgur.com/6VE8nU1.png?1" style="max-width:100%;"></a> <a href="https://camo.githubusercontent.com/f182be3f23b72b98e2b7d3017cf5ae006cc78526/687474703a2f2f692e696d6775722e636f6d2f66716d5854396b2e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/f182be3f23b72b98e2b7d3017cf5ae006cc78526/687474703a2f2f692e696d6775722e636f6d2f66716d5854396b2e706e673f31" alt="Visualization" data-canonical-src="http://i.imgur.com/fqmXT9k.png?1" style="max-width:100%;"></a></p>
<p><a href="https://camo.githubusercontent.com/e954c27cbd3a3a67ea3b8212850a9c2223def479/687474703a2f2f692e696d6775722e636f6d2f5149555870524d2e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/e954c27cbd3a3a67ea3b8212850a9c2223def479/687474703a2f2f692e696d6775722e636f6d2f5149555870524d2e706e673f31" alt="Visualization" data-canonical-src="http://i.imgur.com/QIUXpRM.png?1" style="max-width:100%;"></a> <a href="https://camo.githubusercontent.com/d8255d4d812d52310588121e99ca1b682d16ec56/687474703a2f2f692e696d6775722e636f6d2f487462584b4f362e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/d8255d4d812d52310588121e99ca1b682d16ec56/687474703a2f2f692e696d6775722e636f6d2f487462584b4f362e706e673f31" alt="Visualization" data-canonical-src="http://i.imgur.com/HtbXKO6.png?1" style="max-width:100%;"></a></p>
<h3><a id="user-content-trackers" class="anchor" href="#trackers" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Trackers</h3>
<p>Trackers activity shows the available trackers and if they're turned on or off. There's also a <strong>update interval</strong> setting, which defines how often collected data will be sent to the remote server.
Clicking on a tracker card opens up a settings activity for the given tracker. Long click on a tracker card selects it for visualizing data on the map.</p>
<p><a href="https://camo.githubusercontent.com/5e87533cd1c8b5404cf5ecf24e2e3aa8bc3658c6/687474703a2f2f692e696d6775722e636f6d2f616f67363471692e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/5e87533cd1c8b5404cf5ecf24e2e3aa8bc3658c6/687474703a2f2f692e696d6775722e636f6d2f616f67363471692e706e673f31" alt="Trackers" data-canonical-src="http://i.imgur.com/aog64qi.png?1" style="max-width:100%;"></a> <a href="https://camo.githubusercontent.com/58a16ca00f332a8169f143f44c7557235ab5b75c/687474703a2f2f692e696d6775722e636f6d2f356157576d4a6c2e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/58a16ca00f332a8169f143f44c7557235ab5b75c/687474703a2f2f692e696d6775722e636f6d2f356157576d4a6c2e706e673f31" alt="Trackers" data-canonical-src="http://i.imgur.com/5aWWmJl.png?1" style="max-width:100%;"></a></p>
<h3><a id="user-content-tracker-settings" class="anchor" href="#tracker-settings" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Tracker settings</h3>
<p>All trackers have a set of settings that can be tuned further. These settings are separated in 3 sections:</p>
<ol>
<li>
<p>Sense Interval - Defines how fast new data will be collected. This setting is modifiable only for pull trackers, as push trackers collect data on change.</p>
</li>
<li>
<p>Visualizations - List of possible ways to visualize the collected data. Checking or unchecking the box defines if that visualization will be shown on the <strong>dashboard</strong>.</p>
</li>
<li>
<p>Sensor specific settings</p>
</li>
</ol>
<ul>
<li>Steps tracker has a <strong>daily goal</strong> setting. This is basically how many steps the user wants to make during the day. Going over the goal will send a notification congratulating the user on achieving the goal.</li>
<li>Activity tracker has a <strong>daily goal</strong> setting, showing how many minutes user wants to be active throughout the day. Going over the goal will send a notification congratulating the user on achieving the goal.</li>
<li>Screen tracker has a <strong>daily limit</strong> setting. If screen-on hours go above the limit, <strong>sense</strong> will warn the user that they should be more active throughout the day.</li>
<li>Microphone tracker has a <strong>noise threshold</strong> setting. If noise levels surpass the threshold, user will be alerted and location in which it happened will be recorded.</li>
</ul>
<p><a href="https://camo.githubusercontent.com/cbd4295c1388990ffd25ae6d2d5169a2a986469b/687474703a2f2f692e696d6775722e636f6d2f37626e6a526b652e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/cbd4295c1388990ffd25ae6d2d5169a2a986469b/687474703a2f2f692e696d6775722e636f6d2f37626e6a526b652e706e673f31" alt="Settings" data-canonical-src="http://i.imgur.com/7bnjRke.png?1" style="max-width:100%;"></a> <a href="https://camo.githubusercontent.com/ec15ed06089905c356bc48732243e0f7692950e7/687474703a2f2f692e696d6775722e636f6d2f463643624473712e706e673f31" target="_blank"><img src="https://camo.githubusercontent.com/ec15ed06089905c356bc48732243e0f7692950e7/687474703a2f2f692e696d6775722e636f6d2f463643624473712e706e673f31" alt="Settings" data-canonical-src="http://i.imgur.com/F6CbDsq.png?1" style="max-width:100%;"></a></p>
<h2><a id="user-content-architecture" class="anchor" href="#architecture" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Architecture</h2>
<p>Sense is based around 4 main concepts:</p>
<ol>
<li>Tracker</li>
<li>Visualization Adapter</li>
<li>Model</li>
<li>View Initializer</li>
</ol>
<h3><a id="user-content-tracker" class="anchor" href="#tracker" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Tracker</h3>
<p>Trackers represent the main model of Sense. Each tracker is composed of a <strong>resource</strong>, <strong>accent</strong> and a <strong>theme</strong> which are used for grouping and visual purposes. Besides that, trackers maintain a collection of <strong>attributes</strong> that can be visualized, how these attributes will be visualized and which adapter will be used to adapt/transform the collected data to the expected format.</p>
<h4><a id="user-content-merged-tracker" class="anchor" href="#merged-tracker" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Merged Tracker</h4>
<p>Merged Tracker is a special kind of tracker that’s used to fuse multiple trackers into a single one. Given a visualization type, merged tracker will collect corresponding <strong>models</strong> and join them into one <strong>model</strong> that can be visualized.</p>
<h3><a id="user-content-visualization-adapter" class="anchor" href="#visualization-adapter" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Visualization Adapter</h3>
<p><strong>Visualization Adapters</strong> are responsible for transforming the collected data (instance of SensorData/collection of SensorData instances) into the type that the <strong>View</strong> is expecting. Adapters are meant to be stateless as their only concern is how to transform data from type A to type B.</p>
<h4><a id="user-content-aggregation" class="anchor" href="#aggregation" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Aggregation</h4>
<p>Depending on the <strong>tracker</strong> (more specifically how data is collected), some adapters need to aggregate the data to be able to transform it correctly. This is usually done when adapting data that's received from the server.</p>
<h3><a id="user-content-model" class="anchor" href="#model" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Model</h3>
<p><strong>Model</strong> is a contextual wrapper around the adapted data. This means that models store additional information besides the data that needs to be displayed: e.g. which <strong>tracker</strong> data originated from, or which of the attributes is visualized, etc.
There’s a <strong>model</strong> for each of the visualization types.</p>
<h3><a id="user-content-view-initializer" class="anchor" href="#view-initializer" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>View Initializer</h3>
<p><strong>View Initializers</strong> represent stateless view controllers that are basically the “glue” between <strong>views</strong> and <strong>models</strong>.</p>
<p>Each <strong>View Initializer</strong> goes through the next steps:</p>
<ol>
<li>Construct a <strong>view</strong></li>
<li>Initialize the <strong>view</strong> with the provided data</li>
<li>Setup Update/Clear callbacks</li>
</ol>
<p>From that point on, <strong>views</strong> can be modified only through callbacks mentioned in step 3. Update/Clear callbacks are essentially closures that capture the scope in which <strong>views</strong> are created and initialized.</p>
<p><strong>View Initializers</strong> are “tagged” in two ways:</p>
<ol>
<li>By <strong>model</strong> class type</li>
<li>By visualization type</li>
</ol>
<p>This means that any part of the application can request the correct <strong>view initializer</strong> only by knowing which data should be displayed, or how it should be displayed (visualization type).</p>
<h2><a id="user-content-emotion-sense" class="anchor" href="#emotion-sense" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Emotion Sense</h2>
<p>Another big part of Sense is <a href="http://emotionsense.github.io/index.html">Emotion Sense</a> library. This is an open-source library developed as a part of the EPSRC Ubhave project licensed with the <a href="https://en.wikipedia.org/wiki/BSD_licenses">BSD License</a>.</p>
<p>Emotion sense consists of:</p>
<ol>
<li>
<p>Sensor Manager</p>
<p>The ESSensorManager library provides a uniform and easily configurable access to sensor data, supporting both one-off and continuous sensing scenarios.</p>
</li>
<li>
<p>Sensor Data Manager</p>
<p>The ESSensorDataManager library allows you to format, store and transfer data to your server.</p>
</li>
</ol>
<h3><a id="user-content-sensor-data" class="anchor" href="#sensor-data" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Sensor Data</h3>
<p>Emotion Sense defines "sensor" as any signal that can be unobtrusively captured from the smartphone device. Sense app follows the same convention and principles for adding and managing sensors data.
In SensorManager library you can find two types of sensors:</p>
<p><strong>Pull Sensors</strong></p>
<p>All sensors that the Android OS does not capture data from until requested to do so by an application.</p>
<p><strong>Push Sensors</strong></p>
<p>The Android OS publishes data about particular events that applications can receive, this set of sensors receives this information on behalf of an application.</p>
<h3><a id="user-content-adding-a-sensor" class="anchor" href="#adding-a-sensor" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Adding a Sensor</h3>
<p>To add a new sensor, you need to edit the SensorManager and SensorDataManager libraries. To implement your sensor, you'll need to:</p>
<ol>
<li>Decide whether your sensor is best implemented as a push (AbstractPushSensor), pull (AbstractPullSensor), or environment (AbstractEnvironmentSensor) sensor. This will determine where your code will be implemented and what AbstractSensor type it will inherit from.</li>
<li>Implement the data structure that will store your sensor's data, in the data package. Implement a processor that creates an instance of your sensor's data in the process package.</li>
<li>Implement a default configuration for your sensor in the config package. You will also need to add an entry to getDefaultConfig() in SensorConfig.</li>
<li>The ES Sensor Manager needs to be able to find your sensor. Add an int SENSOR_TYPE and String SENSOR_NAME to SensorUtils. Add an entry for your sensor in SensorEnum.</li>
<li>Add a data formatter to convert your sensor data instance into JSON in the data manager data formatter package.</li>
<li>The data formatter needs to be able to find your formatter. Add an entry in getJSONFormatter() in DataFormatter.java.</li>
</ol>
<p>For more information on how to use and contribute to EmotionSense library, please visit <a href="http://emotionsense.github.io/index.html">Emotion Sense</a>.</p>
